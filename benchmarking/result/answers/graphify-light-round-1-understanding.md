**Repository Understanding Report**

This appears to be a Rust CLI project named/generated around `graphify-light`: a lightweight code graph extractor and query tool. The only inspected artifact was `graph.json` in `.ai/graphify-light`, generated by `graphify-light 0.1.0`.

**Purpose**

The project scans a repository, detects code files, extracts symbols/imports/calls, writes a structured graph artifact, and exposes query surfaces for symbols, callers/callees, related files, imports/exports, and graph stats. It also includes Codex/MCP integration helpers and a benchmarking harness for comparing graph-assisted Codex runs.

**Architecture**

The graph contains 239 nodes and 467 edges across 10 files: 8 Rust source files and 2 Cargo manifests. Relations include `contains`, `imports`, `calls`, and one `imports_from`. The main executable is organized as a small Rust CLI that delegates to focused modules:

- `src/main.rs`: CLI entrypoint using `clap`; dispatches build, query, MCP, and install flows.
- `src/detect.rs`: repository traversal and file filtering. It detects supported languages, skips sensitive/large files, handles shebangs, and returns `DetectedFile` records.
- `src/extract.rs`: graph extraction engine. It uses `tree_sitter`, regex, import/call records, symbol detection, call resolution, import resolution, and fallback heuristic extraction.
- `src/graph.rs`: graph data model and persistence. Defines `Node`, `Edge`, `Graph`, `GraphStats`, and `GraphBuilder`; handles IDs, path normalization, graph read/write, stats, and output paths.
- `src/query.rs`: query API over graph data: `find_symbol`, `get_callers`, `get_callees`, `get_file_symbols`, `search_nodes`, `get_related_files`, `get_imports`, `get_exports`, and stats.
- `src/mcp.rs`: stdio MCP server implementation. It exposes graph build/read/query operations as tools and formats JSON-RPC style responses.
- `src/install.rs`: Codex integration installer. It builds/writes the graph and updates managed blocks in config/`AGENTS.md`; includes tests for managed block replacement.
- `benchmarking/src/main.rs`: separate benchmarking binary that copies repos, runs commands/Codex, parses token usage, compares variants, and emits markdown/round-table reports.
